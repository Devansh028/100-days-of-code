# Database Design Basics

## Primary Key

- Unique identifier for each row

## Foreign Key

- Links two tables

## Index

- Improves read performance
- Slower writes

## Normalization

- Reduce data redundancy
- Use 3NF generally

## When to Denormalize?

- Read-heavy systems
- Performance critical paths

## SQL vs NoSQL

SQL:

- Structured
- ACID
  NoSQL:
- Flexible schema
- High scalability
